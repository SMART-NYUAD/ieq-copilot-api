# DB Benchmark Pack

This folder contains reproducible benchmark queries for sensor_readings and example output/time snapshots.

## Files

- sensor_readings_queries.sql: SQL query pack for manual benchmarking in psql/DB clients.
- run_sensor_readings_benchmark.py: Runs EXPLAIN ANALYZE timings and captures sample outputs.
- sensor_readings_benchmark_latest.json: Most recent benchmark output generated by the Python runner.
- run_meeting_metrics_report.py: Generates meeting-focused results for CO2/PM2.5/multi-metric windows.
- meeting_metrics_results.json: Structured output from meeting report generator.
- MEETING_METRICS_REPORT.md: Meeting-ready markdown summary with timings, outputs, and SQL.

## Run

From the RAG_API_SERVER directory:

```bash
python db_benchmark/run_sensor_readings_benchmark.py
```

Use DATABASE_URL to override the DSN.

```bash
DATABASE_URL='postgresql://admin:secret123@192.168.50.12:5432/campus_iot' \
python db_benchmark/run_sensor_readings_benchmark.py
```

Meeting report run:

```bash
python db_benchmark/run_meeting_metrics_report.py
```

## What this benchmark demonstrates

- Query-shape differences between:
  - JOIN on device(space) + sensor_readings
  - pre-resolved device_id filter with ANY(array)
- Impact of indexes:
  - ix_device_space on device(space)
  - ix_sr_device_metric_ts on sensor_readings(device_id, metric_id, ts desc)

## Current caveat

Timescale hypertables do not support CREATE INDEX CONCURRENTLY, so ix_sr_device_metric_ts must be created without CONCURRENTLY.
